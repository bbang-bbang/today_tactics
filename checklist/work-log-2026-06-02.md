## 2. 선수 배너 인사이트 — 전면 개편
요청이 단계적으로 들어와 다회 반복. 최종 구조: **TOP 퍼포머(종합표) → 심화 인사이트(2×2) → 카드 순위**.

- **TOP 퍼포머 재설계**:
  - 등록 포지션 F/M/D 하드 버킷 → **포지션 무관 통합 리더보드**(백엔드 `top-performers`에 `all` 단일쿼리 신설, 미사용 F/M/D 쿼리 제거).
  - 포지션 칩은 컬럼 교체가 아닌 **행 필터**. 기본 20행 미리보기 + "더 보기" 토글 + 스크롤 캡(헤더 고정)·zebra.
  - **xG 효율 박스를 표 컬럼으로 흡수**(xG·결정력 G−xG·슈팅) → 박스 3→2개, 표 전체너비.
  - **파생 비교점수**: 공격P(골+도움)·창출P((키패스+도움×2)/90)·수비P((태클+인터셉트×1.5+클리어+공중볼승+듀얼승)/90).
  - **부문별 TOP 카드**: 득점·도움·결정력·창출·수비·평점 각 1위를 박스로(현재 필터 집합 기준, 클릭 시 해당 지표 정렬).

- **이중필터 검증**: 리그(재조회) × 포지션 칩(행 필터) 조합 — K2+미드/K1+수비/전체+공격 모두 정상.

## 7. 성능 — 인사이트 "버퍼 현상" 해소
- **진단**: 펼칠 때 6개 엔드포인트 동시 호출 중 **activity(히트맵)만 콜드 ~5s**, 나머지 0.06~0.2s. 인덱스는 충분.
- **진짜 원인**: 부팅 워밍업이 `league=K1`(+구 엔드포인트)만 데움 → 프런트 기본 `year=2026&league=all` 요청은 **항상 캐시 미스(콜드)**.
- **수정**:
  1. 워밍업 URL을 프런트 기본값과 **정확히 일치** + 신규 4종 추가 → 사용자는 항상 캐시 히트.
  2. `loadAll`이 메인(TOP·카드) 렌더 후 심화 패널 로드(`Promise.allSettled().finally`) → 메인 즉시, 히트맵 비차단.
  3. weather: 온도 버킷별 4회 풀스캔 → 단일 GROUP BY 1회(서버 35s 타임아웃 → 0.06s).
  4. activity: 선수당 상관 서브쿼리 제거(상위 8명만 해소).
- **결과**: 전 인사이트 엔드포인트 운영 **0.06~0.08s**.
